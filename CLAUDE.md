# 알바 급여 검증기 — 작업 규칙

이 저장소는 `index.html` 단일 파일로 된 급여 검증 웹앱이에요.
바닐라 JS로 작성됐고, GitHub Pages로 배포돼요. 빌드 과정 없이 파일 하나만 수정하면 돼요.

## 커밋 · 푸시 규칙 (항상 지킬 것)

- 코드를 수정한 뒤에는 **항상 자동으로** `git add` → `git commit` → `git push`까지 한다.
  사용자가 매번 "커밋해줘"라고 말하지 않아도 수정이 끝나면 바로 실행한다.
- 커밋 메시지는 **한국어**로, 무엇을 왜 바꿨는지 한 줄로 명확히 적는다.
  예: `야간수당 기준 시각을 22시에서 23시로 변경`
- 여러 파일을 수정했으면 관련된 변경을 하나의 커밋으로 묶는다.
- 푸시 전에 `git status`와 `git diff`로 의도한 변경만 포함됐는지 확인한다.
- 푸시가 끝나면 커밋 해시와 메시지를 사용자에게 한 줄로 알려준다.

## 하지 말 것

- `git push --force`는 사용하지 않는다 (히스토리 손상 방지).
- 커밋 메시지에 공동 작성자(Co-authored-by)나 도구 서명은 넣지 않는다.
- `index.html` 외의 파일을 임의로 새로 만들지 않는다. 필요하면 먼저 물어본다.
- 사용자 데이터(급여·근무 기록)는 브라우저 localStorage에만 저장되고 저장소에는 올라가지 않는다. 테스트용 실제 데이터를 코드에 하드코딩하지 않는다.

## 코드 구조 메모

- 계산 로직: `compute()` (월 급여), `computeTools()` (실업급여·누적급여·퇴직금)
- 하루 급여: `calcDayPay(entry, wage, sojungH)` — 사업장 실제 방식(개별 주 주휴, 공휴일 유급휴일수당 없음, 실근로 임금+휴일가산만)
- 렌더: `render()`가 전체를 다시 그림. 상태는 `state` 객체 하나에 모여 있고 변경 후 `render()` 호출.
- 저장: `save()` / `load()` — localStorage 키는 `wage-calc-html-v1`.
- 데이터 백업: `exportData()` / `importData()` — JSON 파일 내보내기·불러오기.

## 배포

- GitHub Pages가 `index.html`을 첫 화면으로 서빙한다.
- 푸시 후 몇 분 내 반영된다. 별도 배포 명령은 필요 없다.
